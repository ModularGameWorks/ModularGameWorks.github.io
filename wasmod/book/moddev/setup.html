Mod Project Setup

Now that you have Rust and Cargo installed, you can set up your mod project directory using Cargo.

  1. Create a new library with Cargo, and navigate to that directory (henceforth referred to as the 'mod project directory'). It should be the mod_name you specify, in the directory where you run the command:
  cargo new <mod_name> --lib
  1. Add the WASMod library to your project with Cargo. This should be executed in the mod project directory. Since you're a mod developer, the default features are already set up for you. Hooray!
  cargo add --git https://github.com/ModularGameWorks/wasmod.git

Alternatively, you may choose to add the following line under [dependencies] to the Cargo.toml file in your mod project directory:

  wasmod = { git = "https://github.com/ModularGameWorks/wasmod.git" }
  1. You should implement your mod in src/lib.rs. There are a few important things to note here:

  • src/lib.rs should begin with pub use wasmod::*;. This will export all of the WASMod helper functions, which are necessary for passing structs between a game and its mods.

  • All functions made available by the game developer can be imported in a block like this:

#![allow(unused)]
fn main() {
// This tells the linker to expect these functions from the runtime that 
// runs the WASM code
#[link(wasm_import_module = "host")]
// This tells Rust that these functions will be in the C ABI, which is used
// in WASM
extern "C" {
  // All of the game function declarations will go here, and follow the 
  // following format: 
  fn name(param: type, ...) -> return_type;
}
}
  • Any implementations of hooks defined by the game developer should be functions defined with the following format:
#![allow(unused)]
fn main() {
// This tells the compiler not to mangle the name of this function, so that 
// the mod loader can find it correctly
#[no_mangle]
// This tells Rust to make the function public, and follow the C ABI
pub extern "C" fn name(param: type, ...) -> return_type {
  // TODO: implementation for hook "name" goes here...
}
}
  • You may also choose to organise your mod code into multiple files. In this case you will need to define those files as modules, and export them as well. Suppose we have another file at src/helper.rs. In src/lib.rs, we should include the following lines:
#![allow(unused)]
fn main() {
// specifies the helper.rs file as a module
mod helper;
// re-exports everything that helper.rs exports
pub use helper::*;
}
  1. Now, we can build our mod to WASM. First, we should make sure we have the wasm32-wasi target installed for Cargo. These two commands will result in a <mod_name>.wasm file in target/wasm32-wasi/release.
  rustup target add wasm32-wasi
  cargo build --release --target wasm32-wasi
  1. Finally, you need to set up the mod distribution directory. This will be the set of files someone else needs to run your mod. You should copy in the generated <mod_name>.wasm, along with any resources your mod uses (images, files, etc.). In this directory, you'll need to create a meta.ron file, which will specify some metadata about your mod:
Mod(
  // The name of your mod. Hopefully this field is self explanatory :)
  name: "<mod_name>",
  
  // A list of the names of the mods your mod depends on. These will be loaded
  // prior to your mod. This may be an empty list.
  dependencies: [
    "Dependency 1", 
    "Dependency 2",
  ],
  
  // A list of the names of the hooks that your mod implements. This tells the
  // modloader which hooks to look for in your mod file.
  exports: [
    "hook_one",
    "hook_two",
  ],
  
  // The path from the directory where this file is to your WASM code. If 
  // they're in the same directory, it'll look something like this:
  main_file: "<mod_name>.wasm",
  
  // Any new hooks that your mod defines, which other mods can use. Your mod is
  // expected to make the calls to these hooks, and other mods which use them
  // will be dependent on your mod.
  new_hooks: [
    "my_mod_hook_one",
    "my_mod_hook_two",
  ]
)
  1. Now, with all of that complete, you can finally test your mod! (Note that you should probably test it as you develop, not just when you're finished.) Copy the entire mod distribution directory into the mods directory of the game, and enable it! You can also share the mod distribution directory (probably as a .zip file), so others can use your mod too!
